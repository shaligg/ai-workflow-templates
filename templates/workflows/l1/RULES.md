# Workflow Rules (L1)

Workflow mode: L1 Quick Task.

## Scope

- Implement only the requested small change.
- Keep scope tight and avoid unrelated refactors.
- Do not create feature `design.md` / `tasks.md` by default.

## Execution

- Confirm task scope briefly before coding.
- Add tests when behavior changes.
- Stop after completion and summarize risk.

## Repository Layout

- Use current repository layout as source of truth.
- Do not force `src/`.
- Use actual paths from current repository.
